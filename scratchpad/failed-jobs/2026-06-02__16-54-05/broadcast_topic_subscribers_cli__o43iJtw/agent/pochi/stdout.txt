# Broadcast to Topic Subscribers with the MagicBell CLI ## Background MagicBell lets you fan a single broadcast out to every subscriber of a topic instead of having to enumerate recipients one by one. The recipient form for this is subtle and easy to mis-type: a broadcast targeted at topic subscribers MUST use the nested filter object `{ "topic": { "subscribers": true } }`. Passing the topic key as a bare string (e.g. `{ "topic": "topic-..." }`), or as a top-level recipient field, will be rejected and no notification will be delivered (see MagicBell Discussion #213). This task uses the `magicbell` CLI together with a small REST call to: 1. Authenticate the CLI non-interactively. 2. Upsert two MagicBell users with `run-id`-scoped identifiers and sub-addressed emails. 3. Subscribe both users to the same topic, keyed `topic-cli-${ZEALT_RUN_ID}`. 4. Create a broadcast tagged with that topic and targeting every subscriber of the topic. 5.
