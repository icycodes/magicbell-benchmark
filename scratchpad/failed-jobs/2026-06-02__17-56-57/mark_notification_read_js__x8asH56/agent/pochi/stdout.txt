# Mark a MagicBell Notification as Read with the magicbell-js User Client ## Background MagicBell distinguishes between two SDK surfaces in `magicbell-js`: the **Project Client** (server-side admin, authenticated with the Project JWT) and the **User Client** (acts on behalf of a single end user, authenticated with a short-lived HS256 User JWT). End users can list, read, archive, etc., their own notifications through the User Client. In this task you will combine both clients: use the Project Client to upsert the user and broadcast a notification, mint a User JWT for that user, then use the User Client to find the seeded notification and mark it as read. The resulting notification id must be persisted so the verifier can confirm the read state via the MagicBell REST API. ## Requirements * Work inside a Node.js project under `/home/user/myproject`. * Install and use the modern `magicbell-js` SDK (both `magicbell-js/project-client` and `magicbell-js/user-client`) and `jsonwebtoken` for HS256 signing. * The script must: 1. Compute the per-run recipient email by splitting `MAGICBELL_EMAIL` at `@` into `<local>@<domain>` and forming the plus-addressed mailbox `<local>+mark-read-js-<run-id>@<domain>`. 3. Initialise the Project Client from `magicbell-js/project-client` with `token: process.env.MAGICBELL_PROJECT_TOKEN` and upsert a MagicBell user via `users.saveUser({...})` with: * `externalId`: `user-mark-read-js-<run-id>` * `email`: the plus-addressed email above * `firstName`: `MarkRead` * `lastName`: `JS-<run-id>` 4. Send a broadcast targeting that user (by `externalId`) via `broadcasts.createBroadcast({...})` with: * `title`: `Mark Read JS - <run-id>` * `content`: a short body (e.g., `Notification for mark-read JS run <run-id>.`) * `recipients`: `[{ externalId: 'user-mark-read-js-<run-id>' }]` 5. Mark that notification as read by calling the User Client's mark-read method on `client.notifications` with the notification id. 9. The underlying REST call is `POST /notifications/{notification_id}/read`; consult the official `magicbell-js` user-client docs for the exact JavaScript method name. * Build the User JWT payload as documented for MagicBell User Auth: `{ user_email, user_external_id, api_key }`, signed with `MAGICBELL_SECRET_KEY` using `HS256` and `expiresIn: '1y'`. * Pull the per-run identifiers from `ZEALT_RUN_ID`; do not hardcode them. ## Acceptance Criteria * Project path: /home/user/myproject * Ensure the script is executed end-to-end and the artifacts exist. * Log file: /home/user/myproject/output.log * The log file must contain the notification id in the format: `Notification ID: <notification_id>`. * The MagicBell user `user-mark-read-js-${run-id}` (where `run-id` is read from `ZEALT_RUN_ID`) must exist on the project. * The notification referenced by `<notification_id>`, when fetched as that user from `GET https://api.magicbell.com/v2/notifications/<notification_id>` with `Authorization: Bearer <freshly-minted user JWT>`, must: * Return HTTP status 200. * Have `title` equal to `Mark Read JS - ${run-id}`. * Have a non-null `read_at` timestamp (indicating it was marked as read by the User Client).
