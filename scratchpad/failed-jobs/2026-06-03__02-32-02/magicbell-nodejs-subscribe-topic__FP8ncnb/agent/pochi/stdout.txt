**Key findings during implementation:**
- The MagicBell v2 SDK `ProjectClient` requires a Project JWT obtainable only from the dashboard, so the v1 REST API (`X-MAGICBELL-API-KEY` + `X-MAGICBELL-API-SECRET`) was used for user creation
- The v1 `/subscriptions` endpoint requires `X-MAGICBELL-USER-HMAC` as an HMAC-SHA256 digest encoded in **base64** (not hex) — mixing it with a Bearer JWT causes a 500 error
- The subscription body must use `{ subscription: { topic, categories: [{ slug: "*" }] } }` format
