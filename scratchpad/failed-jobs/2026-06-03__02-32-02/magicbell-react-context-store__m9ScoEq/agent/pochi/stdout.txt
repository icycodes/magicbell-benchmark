## What was built

### Project structure
- **Framework**: Vite + React
- **SDK**: `@magicbell/react-headless` v5.5.1

### Key files

**`src/CustomBadge.jsx`** — Custom badge component using the `useNotifications` hook:
```jsx
import { useNotifications } from '@magicbell/react-headless';

function CustomBadge() {
  const store = useNotifications();
  const unreadCount = store ? store.unreadCount : 0;

  return (
    <div className="custom-badge">
      <span>Unread count: {unreadCount}</span>
    </div>
  );
}
```

**`src/App.jsx`** — Root component wrapping with `MagicBellProvider`:
```jsx
import { MagicBellProvider } from '@magicbell/react-headless';

function App() {
  return (
    <MagicBellProvider apiKey={MAGICBELL_API_KEY} userEmail={MAGICBELL_USER_EMAIL}>
      <CustomBadge />
    </MagicBellProvider>
  );
}
```

**`.env`** — Credentials configured via Vite env vars:

**`vite.config.js`** — Port set to 3000 with `server.port: 3000`.

### How it works
1. `MagicBellProvider` initializes the notification store collection with the API key and user email
2. `useNotifications()` inside `CustomBadge` subscribes to the default notification store
3. The store's `unreadCount` is rendered as `Unread count: X` — defaulting to `0` while the store initializes, then updating to the actual count once fetched from the MagicBell API

The app is live at **http://localhost:3000** and renders `Unread count: X` with the real-time unread count from the MagicBell notification store.
